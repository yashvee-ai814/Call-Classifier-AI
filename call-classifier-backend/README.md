# Call Classifier — Backend

A FastAPI service that classifies phone call transcripts by sending them to a locally-running LLM via [Ollama](https://ollama.com).

---

## How it works

1. The frontend (or any HTTP client) sends a POST request with a raw call transcript.
2. The backend builds a structured prompt and forwards it to Ollama.
3. Ollama runs the LLM locally and returns a generated JSON response.
4. The backend parses the response and returns a `reason` and `category` to the caller.

---

## Prerequisites

| Tool | Purpose | Install |
|------|---------|---------|
| Python ≥ 3.11 | Runtime | [python.org](https://www.python.org/downloads/) |
| [uv](https://github.com/astral-sh/uv) | Package manager | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| [Ollama](https://ollama.com) | Run LLMs locally | [ollama.com/download](https://ollama.com/download) |

After installing Ollama, pull the model the app expects:

```bash
ollama pull gpt-oss:120b-cloud
```

> You can override the model via the `OLLAMA_MODEL` environment variable (see Configuration below).

---

## Running locally

```bash
# 1. Enter this directory
cd call-classifier-backend

# 2. Create a virtual environment and install all dependencies
uv sync

# 3. Make sure Ollama is running in another terminal
ollama serve

# 4. Start the API server with hot-reload enabled (reloads on file changes)
uv run uvicorn main:app --reload
```

The API is now live at **http://localhost:8000**.  
Interactive docs (auto-generated by FastAPI): **http://localhost:8000/docs**

---

## Running with Docker

```bash
# Build the image
docker build -t call-classifier-backend .

# Run the container
# OLLAMA_HOST points to Ollama running on your host machine
docker run -p 8000:8000 \
  -e OLLAMA_HOST=http://host.docker.internal:11434 \
  call-classifier-backend
```

## API Endpoints

### `POST /classify/`

Classify a call transcript.

**Request body**
```json
{
  "transcript": "Hello, I'd like to check the balance on my pension account."
}
```

**Response**
```json
{
  "reason": "Customer wants to check their pension account balance",
  "category": "account_status"
}
```

**Category values**

| Value | Meaning |
|-------|---------|
| `inquiry` | General question about products or services |
| `advice_request` | Customer asking for financial advice |
| `complaint` | Customer raising a problem or complaint |
| `account_status` | Checking balance, statement, or account details |
| `transaction` | Making or querying a transfer, payment, or trade |
| `other` | Anything that does not fit the above |

## Configuration

Set these environment variables to customise behaviour:

| Variable | Default | Description |
|----------|---------|-------------|
| `OLLAMA_HOST` | `http://localhost:11434` | Base URL of the Ollama server |
| `OLLAMA_MODEL` | `gpt-oss:120b-cloud` | Name of the model to use |
